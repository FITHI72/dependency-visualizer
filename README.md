Dependency Visualizer
## 1. Общее описание

Dependency Visualizer — это консольный инструмент для анализа зависимостей программных пакетов.
Он строит граф зависимостей и позволяет вычислять корректный порядок их загрузки (топологическую сортировку), а также обнаруживать циклы.

Поддерживаются режимы:

graph — вывод графа зависимостей;

order — вычисление порядка загрузки зависимостей.

Программа умеет работать:

с локальными тестовыми репозиториями (--mode test);

частично — с NuGet API (прямые зависимости).

## 2. Описание функций и настроек
Пакет config

Отвечает за обработку параметров командной строки.

Параметры:
Флаг	     Пример	             Описание
--name	   A	                 Имя анализируемого пакета
--url	     ./repo.txt	         Путь/URL репозитория
--mode	   test	               Режим работы (test/off/offline/remote)
--version	 1.0.0	             Версия пакета
--op	     graph / order	     Тип операции

Пакет validate

Проверяет корректность входных параметров:

существование файла в test-режиме,

корректность URL при remote,

формат версии (semver),

корректность имени пакета.

Пакет fetch

Функции:

LoadTestRepo(path string)

Загружает зависимости из текстового файла вида:

A: B C
B: D
C: D E
D:
E:


Возвращает map[string][]string.

Пакет graph
NewGraph() - Создаёт новый пустой граф.

AddDependency(pkg, dep) - Добавляет зависимость pkg → dep.

BuildDFS(start, fetchDeps) - Строит полный граф зависимостей, начиная от пакета start.

PrintGraph() - Выводит граф в консоль.

LoadOrder(start string) - Выполняет топологическую сортировку: если циклов нет → возвращает корректный порядок загрузки, если цикл есть → возвращает список узлов цикла.

## 3. Команды для сборки проекта и запуска тестов
Сборка
go build -o depvis .

Запуск в тестовом режиме

Пример файла repo.txt:

A: B C
B: D
C: D E
D:
E:

Запуск — вывод графа
go run . --name A --url ./repo.txt --mode test --version 1.0.0 --op graph

Запуск — порядок загрузки
go run . --name A --url ./repo.txt --mode test --version 1.0.0 --op order

## 4. Примеры использования
Пример 1 — граф зависимостей

Команда:

go run . --name A --url repo.txt --mode test --version 1.0.0 --op graph


Вывод:

--- Граф зависимостей ---
A -> [B C]
B -> [D]
C -> [D E]
D -> []
E -> []

Пример 2 — порядок загрузки

Команда:

go run . --name A --url repo.txt --mode test --version 1.0.0 --op order


Вывод:

Порядок загрузки зависимостей:
D → E → B → C → A


Пример 3 — обнаружение цикла

Файл:
A: B
B: C
C: A

Команда:
go run . --name A --url repo.txt --mode test --version 1.0.0 --op order

Вывод:
⚠ Обнаружен цикл в зависимостях: A B C
